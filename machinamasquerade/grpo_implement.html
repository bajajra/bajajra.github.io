GRPO: Huggingface Implementation#

GRPO in a Nutshell#

GRPO maximises the advantage of each sampled completion without a learned critic.
For a prompt (x) we draw a group of (G) completions \(\{y_i\}\), compute task reward \(r_i\) for each, and z-score normalise

\[\begin{split} \begin{aligned} \tilde r_i &= \frac{r_i - \mu_r}{\sigma_r}, \\[6pt] \mu_r &= \frac{1}{G}\sum_{j=1}^{G} r_j, \\[6pt] \sigma_r &= \sqrt{\frac{1}{G}\sum_{j=1}^{G} \bigl(r_j - \mu_r\bigr)^2}. \end{aligned} \end{split}\]

The objective is

\[ \mathcal{L}(\theta)= \sum_{i=1}^{G} \min\!\Bigl( r_{i,t}(\theta)\,\tilde r_i,\, \operatorname{clip}\bigl(r_{i,t}(\theta),\,1-\varepsilon,\,1+\varepsilon\bigr)\,\tilde r_i \Bigr) -\beta\,D_{\mathrm{KL}}\bigl(\pi_{\theta}\,\Vert\,\pi_{\text{ref}}\bigr). \]

where
$\( r_{i,t}(\theta)=\frac{\pi_{\theta}(y_i \mid x)}{\pi_{\text{ref}}(y_i \mid x)}. \)$ Because there is no critic head, you save 30â€“40â€¯% GPU memory and can scale to larger models.

Designing a Reward Function#

A reward callback receives a batch of {prompt, completion, â€¦} dictionaries and returns an array of floats.
Below is a minimal example that penalises long outputs and rewards wellâ€‘formed sentences:

import numpy as np
from nltk.tokenize import word_tokenize

def simple_reward(samples):
    out = []
    for s in samples:
        length_penalty = -0.001 * max(0, len(word_tokenize(s["completion"])) - 512)
        end_bonus      = 1.0 if s["completion"].strip().endswith(".") else 0.0
        out.append(length_penalty + end_bonus)
    return np.array(out, dtype=np.float32)

Normalise raw rewards to roughly \(\\([-5,5]\\)\) before the zâ€‘score step; extreme values slow convergence.

WiringÂ GRPOTrainer#

from transformers import AutoTokenizer, AutoModelForCausalLM
from trl import GRPOConfig, GRPOTrainer

model_name = "Qwen/Qwen2.5-3B-Instruct"

tokenizer = AutoTokenizer.from_pretrained(model_name, use_fast=True)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    load_in_4bit=True,      # quantised weights
    device_map="auto")      # spread over available GPUs

config = GRPOConfig(
    group_size                = 4,      # completions per prompt
    beta                      = 0.05,   # KLâ€‘penalty factor
    learning_rate             = 5e-6,
    mini_batch_size           = 4,
    batch_size                = 128,
    target_kl                 = 0.1,    # autoâ€‘tune beta
    gradient_accumulation_steps = 4,
    reward_fn                 = simple_reward,
)

trainer = GRPOTrainer(
    model         = model,
    ref_model     = None,       # frozen copy instantiated internally
    args          = config,
    tokenizer     = tokenizer,
    train_dataset = ds,
)

trainer.train()

Key points:

  • ref_model=None duplicates the initial weights internallyâ€”saves RAM.

  • group_sizeÂ =Â 4â€“8 is a sweet spot: lower variance without exploding memory.

  • If actual KL rises above target_kl, the trainer automatically scales beta.

Correlating Theory with Code#

  • group_size in the config is \(\\(G\\)\) in the objective.

  • beta multiplies the KL term.

  • Your reward_fn provides \(\\(r_i\\)\).

  • TRL normalises rewards internally:

    advantage = (r - r.mean()) / (r.std() + 1e-8)

    matching the \(\\(\\tilde r_i\\)\) definition.

  • The minÂ +Â clip logic is executed in a fused CUDA kernel for speed.
